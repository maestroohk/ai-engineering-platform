## 1. Objective

Populate the M2.1 `AppTopBarSlot`
with the **top bar**, the
**breadcrumb**, and the **page
header integration** the design
system already supports. The
session composes the M1.2
primitives (`AppToolbar`,
`AppPageHeader`, `AppStack`,
`AppButton`, `AppIconButton`,
`AppAvatar`, `AppBadge`,
`AppStatusDot`) and the M2.2
registry's `RouteMetadata.Parent`
chain to render the breadcrumb.

The session ships:

- The `AppTopBar` component (the
  real top bar that replaces
  M2.1's `AppTopBarSlot`).
- The `AppBreadcrumb` component
  (composed into `AppPageHeader`'s
  `Breadcrumbs` parameter, which
  was an M1.2 placeholder).
- The theme toggle moved into
  the top bar's `Trailing` slot
  (the M1.1 chrome had the theme
  toggle in the footer; M2.3
  relocates it).
- The user avatar in the top
  bar's `Trailing` slot.
- bUnit tests for the two new
  components.

## 2. Why It Is Separate

M2.3 is a distinct slice from
M2.1 and M2.2 because:

- The top bar is a **horizontal
  region** of the shell, not a
  vertical one. The layout
  composition (M2.1) does not
  care what the top bar
  contains; the top bar's
  content is its own concern.
- The breadcrumb depends on the
  M2.2 registry's `Parent` chain.
  Shipping the breadcrumb in
  M2.2 would couple the slice to
  a data source the slice does
  not own. Shipping it in M2.3
  gives the breadcrumb a stable
  data source.
- The page header integration
  is a cross-cutting concern
  (every page in M3+ uses it).
  Shipping it in M2.3 means
  every later slice composes
  the page header; the slice
  is the right home for the
  "every page has a page header"
  rule.
- The slice is small and
  independently deliverable:
  two new components, the
  theme toggle relocation,
  bUnit tests. The slice ends
  when the top bar renders,
  the breadcrumb follows the
  route's `Parent` chain, and
  the `AppPageHeader.Breadcrumbs`
  parameter is no longer a
  placeholder.

## 8. Implementation Order

The 13-step task lifecycle
(`.ai/workflows/progressive-coding.md`)
governs every step. The M2.3
implementation is small and
independently deliverable; the
session follows the order below.

1. **Plan approval.** The user
   signs off § 17 of this plan.
2. **Branch off the M2.2 closeout.**
   `feature/m2-3-topbar-breadcrumbs`
   branches from the M2.2 closeout
   commit on
   `feature/m2-2-navigation-registry-sidebar`.
3. **Read every M1.2 / M2.1 / M2.2
   artefact.** M2.3 composes the M1.2
   primitives (`AppToolbar`,
   `AppPageHeader`, `AppStack`,
   `AppButton`, `AppIconButton`,
   `AppAvatar`, `AppBadge`,
   `AppStatusDot`); consumes the M2.2
   `INavigationRegistry`; replaces the
   M2.1 `AppTopBarSlot` placeholder.
4. **`AppTopBar` component.** In
   `src/AiEng.Platform.App/Components/Shell/AppTopBar.razor`
   (+ `.razor.css`). Two
   `AppStack` slots (`Leading` and
   `Trailing`) wrap `AppPageHeader` for
   the title (Leading) and the theme
   toggle + user avatar (Trailing).
5. **`AppBreadcrumb` component.** In
   `src/AiEng.Platform.App/Components/Navigation/AppBreadcrumb.razor`
   (+ `.razor.css`). Injects
   `INavigationRegistry`; reads the
   current route's `Parent` chain
   (from `NavigationManager.Uri` +
   `INavigationRegistry.FindByHref` +
   the recursive
   `ChildrenOf`). Renders one
   `AppNavItem` (or a styled breadcrumb
   link) per parent, separated by an
   `aria-hidden="true"` separator.
6. **Theme toggle relocation.** Move
   the existing M1.1 theme toggle
   into `AppTopBar`'s `Trailing` slot.
   The toggle persists the choice to
   `localStorage["app-theme"]` per the
   M1.1 chrome behaviour.
7. **User avatar slot.** Add the
   user avatar in `AppTopBar`'s
   `Trailing` slot. M2.3 ships the
   slot; the avatar is a placeholder
   (the M3+ session replaces it with
   the real user identity surface).
8. **Page header integration.** In
   `src/AiEng.Platform.App/Components/Pages/*.razor`,
   compose `AppPageHeader` with the
   `Breadcrumbs` parameter (the M1.2
   placeholder) for every page that
   uses `AppPageHeader`. The
   `Breadcrumbs` is bound to
   `AppBreadcrumb` for the current
   route's parent chain.
9. **Replace `AppTopBarSlot` with
   `AppTopBar`** in
   `src/AiEng.Platform.App/Layouts/AppLayout.razor`.
10. **bUnit tests.** 3 new test files

    (4–6 tests). Total: 18–24 new
    tests.
11. **Architecture test (optional,

    `Breadcrumb_Follows_Registry_Parent_Chain`
