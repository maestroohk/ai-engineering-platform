## 1. Objective

Close the M2 acceptance criteria
for the application shell by
exercising every route, every
viewport down to 1280x720, and
every keyboard interaction. The
M2.5 slice is the M2.5 DoD
checkpoint: it does not introduce
new product features; it makes
the M2.1 / M2.2 / M2.3 / M2.4
deliverables usable, accessible,
and consistent across the routes
they own.

The M2.5 work has three surfaces:

- **Empty routes** — every route
  in `Components/Pages/` that does
  not have a data-owning section
  renders an `AppEmptyState` that
  names the page and links back to
  `/design-system`. The M2.1
  `AppEmptyState` primitive is the
  contract.
- **Responsive matrix** — the
  layout is usable at every
  viewport down to 1280x720 (per
  ADR-005). Below 1280x720 the
  sidebar collapses to an icon
  rail; the top bar remains
  horizontal; the content area
  scrolls vertically.
- **Accessibility audit** — the
  full keyboard smoke test passes
  on every route; the axe-core
  audit reports zero critical or
  serious violations; the
  `aria-current="page"` invariant
  holds on every navigation; the
  focus ring is visible on every
  focusable element.

## 4. Definition of Done

- Every route in
  `Components/Pages/` reaches an
  `AppEmptyState` (or its
  data-owning equivalent for the
  routes that have one).
- The layout is usable at 1280x720
  (per ADR-005). The sidebar
  collapses to an icon rail below
  1280px; the top bar remains
  horizontal; the content area
  scrolls vertically.
- The keyboard smoke test passes
  on every route: Tab through the
  shell, every interactive element
  is reachable, focus is visible,
  Enter activates the focused
  element, Escape closes any open
  menu or modal.
- The `aria-current="page"`
  invariant holds on every
  navigation: the current route's
  nav item has `aria-current="page"`
  and the breadcrumb's last
  segment has `aria-current="page"`.
- The axe-core audit reports
  zero critical or serious
  violations on every route.
- `dotnet build` → 0 warnings, 0
  errors.
- `dotnet test` → all bUnit
  tests pass (M2.5 tests add to
  the count; the existing
  `Pages_AreReachable_Through_Registry`
  and
  `Pages_Resolve_State_Through_Reader`
  architecture tests remain
  green).
- `dotnet format
  --verify-no-changes` → clean.
- `npm run css:build` → clean.
- The app starts; every route
  returns 200; the visual smoke
  test passes at 1280x720, 1440x900,
  and 1920x1080.
